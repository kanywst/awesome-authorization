# Contribution Guidelines

Thank you for your interest in contributing to Awesome Authorization!

## Adding an Entry

Please ensure your pull request adheres to the following guidelines:

- Search previous suggestions before making a new one to avoid duplicates.
- Make an individual pull request for each suggestion.
- Use the following format: `[Name](link) - Description.`
- Descriptions should be clear, concise, and end with a period.
- New categories or improvements to the existing categorization are welcome.
- Check your spelling and grammar.
- The pull request should have a useful title and include a link to the resource and why it should be included.

## Updating an Entry

- If you find a dead link, please submit a pull request to remove or update it.
- If a project has been renamed or moved, please update the link.

## Quality Standards

To be included, a resource should meet at least one of the following criteria:

- Actively maintained (for tools and frameworks)
- Provides unique insight or information (for articles and talks)
- Widely referenced in the authorization community
- Represents a notable implementation or case study

## Code of Conduct

Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.
